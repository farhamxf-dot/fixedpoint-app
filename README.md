# Fixed Point Predictor — fixedpoint-app

This Streamlit application trains and evaluates neural models that predict fixed points for a variety of dynamical problems. The app exposes many model and training controls via a sidebar and supports loading/saving PyTorch checkpoints.

## Key Features
- Interactive Streamlit UI to configure LSTM, attention, convolutional, and transformer components.
- Train or load models (`.pth` checkpoints) and run a comprehensive testing suite (Volterra, logistic systems, complex function tests).
- Ensemble option combining neural networks with classical ML models.
- Regenerates or uses CSV dataset `functions_fixed_points_progressive.csv` for experiments.
- Renders LaTeX math in text outputs and markdown (see LaTeX section).

## Applications

This work and the `fixedpoint-app` implementation are useful in a range of practical and research settings:

- **Accelerating numerical solvers:** Provide learned initial guesses or direct fixed-point estimates to reduce iterations in root-finding and fixed-point iterations.
- **Dynamical systems analysis:** Rapidly evaluate steady states across parameter sweeps and support bifurcation exploration without full numerical integration.
- **Control systems & robotics:** Estimate equilibrium points for feedback controllers or model-predictive control that require fast steady-state predictions.
- **Inverse problems & parameter estimation:** Use predicted fixed points as targets or constraints when fitting parametric models to observed steady-state data.
- **Engineering & physics simulations:** Serve as a lightweight surrogate for expensive iterative steady-state simulations in design loops.
- **Signal processing & time-series analysis:** Detect steady-state responses or equilibria in sampled signals when analytic solutions are unavailable.
- **Education & visualization:** Interactive demos (via Streamlit) to teach fixed-point concepts, illustrate bifurcations, and visualize predictions with LaTeX-rendered math.

Programmatic inference example (load checkpoint and run a prediction):

```py
import torch
from models import EnhancedCombinedModel

# Load checkpoint
chk = torch.load('fixed_point_model.pth', map_location='cpu')
cfg = chk.get('config', {})

# Build model with saved config (example — adapt to your saved config)
model = EnhancedCombinedModel(
    n_points=cfg.get('n_points', 41),
    k=cfg['oscillator_params']['k'],
    s=cfg['oscillator_params']['s'],
    lstm_hidden_size=cfg['lstm_params']['hidden_size'],
    lstm_num_layers=cfg['lstm_params']['num_layers'],
    lstm_dropout=cfg['lstm_params']['dropout'],
    conv_channels=cfg['conv_params']['channels'],
    kernel_size=cfg['conv_params']['kernel_size']
)
model.load_state_dict(chk['state_dict'])
model.eval()

# Example input: 1 x n_points x 1 tensor of sampled function values
import numpy as np
sample = np.linspace(-1, 1, model.n_points)  # replace with real samples
x = torch.tensor(sample.reshape(1, -1, 1)).float()
with torch.no_grad():
    y_pred = model(x)
print('Predicted fixed point:', y_pred.numpy())
```

## Run the app

From the repository root or the `fixedpoint-app` folder run:

```bash
cd fixedpoint-app
streamlit run main_app.py
```

When the app starts it will open a local web UI (Streamlit will print the local URL to the console). The app will detect `cuda` if available and use it for training, otherwise it runs on CPU.

## LaTeX support (math rendering)

The app uses Streamlit's markdown rendering which supports MathJax-style LaTeX. You can include inline math using single dollar signs and display math using double-dollar signs:

- Inline example: $x^2 + 2x + 1$
- Display example:

$$
f(x) = x^2 + 2x + 1
$$

Place math in any markdown text shown by the app (for example, add it to text displayed via `st.markdown()`/`st.write()` if you extend the app). Streamlit will render inline `$...$` and block `$$...$$` LaTeX automatically.

## Typical workflow
1. Configure model architecture and training parameters in the sidebar.
2. If you don't have `functions_fixed_points_progressive.csv`, the app can generate it (choose rows / points in the sidebar).
3. Click **Run Training** to train a neural model (or choose **Load saved model** to evaluate existing checkpoints).
4. After training the model is saved as a `.pth` checkpoint you can reload from the sidebar.
5. Use the **Testing Options** panel to run the comprehensive testing suite or specific tests (Volterra, logistic, etc.).

## Troubleshooting
- Missing dependency errors: ensure the virtual environment is activated and `pip install -r requirements_streamlit_ml.txt` completed successfully.
- CUDA not available: training will automatically fall back to CPU if no GPU is detected.
- Model load errors: if a checkpoint was saved with a different architecture/config, load with caution — the app tries `strict=False` when loading state dictionaries and will report missing/unexpected keys in the UI.

# Fixed Point Prediction Using a Hybrid Neural Network with Chaotic Oscillators

This repository contains the official source code for the paper, "Fixed Point Prediction Using a Hybrid Neural Network with Chaotic Oscillators" by Farshid Khojasteh, Sasan Soltani, Majid Alavi, and Majid Haghverdi.

We present a novel hybrid neural architecture that directly predicts fixed points from function samples, eliminating the need for traditional iterative methods. Our model uniquely combines Bidirectional LSTMs for global function understanding, Convolutional Networks for local feature extraction, and an innovative array of chaotic oscillators for non-linear feature transformation. This approach achieves state-of-the-art performance, significantly outperforming classical methods and standard deep learning baselines, and shows remarkable generalization to unseen problem classes like Volterra integral equations.

## Key Features

*   **Hybrid Architecture:** Synergizes three distinct branches (Bi-LSTM, 1D-CNN, and Chaotic Oscillators) to create a rich, comprehensive feature representation.
*   **Direct Prediction:** Frames fixed-point computation as a regression task, learning a direct mapping from function samples to the fixed point in a single forward pass.
*   **State-of-the-Art Performance:** Achieves a Mean Squared Error (MSE) of 2.81 × 10⁻⁵ and an R² of 0.986 on a diverse synthetic dataset.
*   **Exceptional Generalization:** Accurately predicts fixed points for Volterra integral equations and dynamical systems without retraining, demonstrating a learned abstract understanding of the mathematical concept.
